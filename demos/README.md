# pxcli feature demos

Each directory below showcases one pxcli capability with a real, runnable
script and its committed output. All demos assume a `pxcli` binary is
built (`go build -o /tmp/pxcli-main ./cmd/pxcli` from the repo root) and
adjust the `pxcli()` alias / `demo.pxs` invocation path as needed — they
were generated with the binary at `/tmp/pxcli-main` (or `/tmp/pxcli-underlay`
for the reference-underlay demo, see below).

| Demo | Feature | Script | Output |
| --- | --- | --- | --- |
| [01-batch-script](01-batch-script) | Batch/script execution | `demo.pxs` (146 lines, one `script` call) | `demo.png` — a shaded gem drawn per-pixel |
| [02-palette](02-palette) | Named palette management | `demo.sh` | `demo.png` — a torch icon built from `p:<index>` references |
| [03-shapes-and-dither](03-shapes-and-dither) | Shapes + dither shading | `demo.pxs` | `demo.png` — a glossy orb using `circle`/`ellipse` fills and `dither_fill` |
| [04-regions-and-symmetry](04-regions-and-symmetry) | copy/paste/move/mirror | `demo.sh` | `demo.png` — a symmetric potion bottle, only the left half hand-drawn |
| [05-layers](05-layers) | Layers | `demo.sh` | `demo.png` — background/shadow/sprite composited non-destructively |
| [06-frames-and-sprite-sheet](06-frames-and-sprite-sheet) | Frames + `frame_ghost` + `export_sheet` | `demo.sh` | `sheet.png` (3-frame squish-bounce cycle) + `ghost-sample.txt` (onion-skin text grid) |
| [07-blend-and-inspect](07-blend-and-inspect) | `blend` + `inspect` utilities | `demo.sh` | `demo.png` (hand-blended AA edges) + `inspect-output.txt` (text-grid dump) |
| [08-reference-underlay](08-reference-underlay) | `import_reference` + `export_debug` | `demo.sh` | `plain-export.png` vs `debug-export.png` vs the traced `reference-sketch.png` |

## Regenerating a demo

Each `demo.sh` is a standalone shell script: `./demo.sh <socket-path> [outputs...]`
(see the script header for its exact positional arguments). `demo.pxs`
files are run with `pxcli script <file>` directly after `pxcli start`.

## Note on 08-reference-underlay

`import_reference`/`export_debug` are not on `main` yet — they land in
[PR #8](https://github.com/timlevett/pixel-art-cli/pull/8) (in review,
pending a windowed-build confirmation; see that PR for details). This
demo was generated by building the `feature/pxcli-reference-underlay`
branch directly rather than `main`. Once PR #8 merges, `demo.sh` will run
against the same `pxcli` binary as every other demo.
